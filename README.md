# To-Do List Git Collaboration Practice

This is a starter project for practising Git and GitHub collaboration. The HTML and CSS are already provided. Interns should complete the JavaScript functionality in `script.js` by working in separate feature branches and pull requests.

## Features to Complete

### Intern 1

- Add new tasks
- Prevent empty tasks from being added
- Edit existing tasks

### Intern 2

- Delete tasks
- Mark tasks as completed
- Filter tasks by All, Pending, and Completed

Task data does not need to remain after the page is refreshed.

## Git Workflow

For every feature:

1. Pull the latest code from the `main` branch.
2. Create a separate feature branch.
3. Complete only the assigned feature.
4. Commit the changes using a meaningful commit message.
5. Push the branch to GitHub.
6. Create a pull request.
7. Ask the other intern to review the pull request.
8. Resolve the review comments, if any.
9. Merge the pull request only after approval.
10. Pull the updated `main` branch before starting the next feature.

## Branch Name Examples

- `feature/add-task`
- `feature/edit-task`
- `feature/delete-task`
- `feature/complete-task`
- `feature/task-filters`

## Commit Message Examples

- `feat: add task creation functionality`
- `feat: add delete task functionality`
- `fix: prevent empty tasks`
- `style: improve completed task appearance`

## Important Rules

- Do not push directly to the `main` branch.
- Create one branch for each feature.
- Create one pull request for each feature.
- Do not include unrelated changes in the same branch.
- Review each other's pull requests before merging.
- Make sure your branch is updated with the latest `main` branch.

## Running Locally

Open `index.html` in a browser. No build tools are required.
